# 目标
分析每一个文件的作用，并在目标的输出目录中输出每个文件的分析html与入口的总结html
# 要求
1. 不需要分析目标目录下的文件
2. 不需要分析.skills目录下的文件
3. 除了上述黑名单外，每个文件都需要分析并形成分析html，任何类型都需要分析
4. 针对单个文件的分析结果一定要详细，包括但不限于整个文件的作用，每个方法的作用，参数及其含义，可能调用的地点。 
5. 针对单个文件的分析，内部在页面的右边一定要专门有一块区域用来讲解该文件涉及到的专业术语并且用通俗的大白话总结一下这个分析文档
6. 入口html，不但可以索引到刚才创建的html，内部的内容还需要能够串联起文件的关系。相当于一个缩略版的总结文档。还需要加入知识图谱，展示每个文件的关系


# 工作流
## 确定输出的目标目录
查看[config.properties](../scripts/config.properties)获取其中的struct.root属性的参数值 
在项目的根目录中有和上述参数值相同的目录，如果没有，请创建一个。并将其作为目标目录。 
## 创建analysis目录
在提取出的目标目录下创建名为analysis目录
## 开始分析
### 流程
1. 遍历项目的所有文件
2. 分析每一个文件的作用，并且在analysis目录下创建每一个文件的分析结果的html。html文件的相关名称要参考原本的路径结构，包括分包也要参考原来的路径结构
3. 最后创建一个入口html，不但可以索引到刚才创建的html，内部的内容还需要能够串联起文件的关系。相当于一个缩略版的总结文档。
