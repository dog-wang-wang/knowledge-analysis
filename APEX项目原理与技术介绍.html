📚 APEX项目原理与技术介绍

自适应个性化知识图谱摘要系统 - 核心技术解析

🎯 项目概述

APEX (Adaptive and Extreme Summarization)
APEX是一个用于个性化知识图谱摘要的系统。它的核心思想是：不同的用��有不同的查询偏好，系统根据用户的历史查询日志，自动构建一个"小图谱"（摘要），这个小图谱只包含用户最可能关心的K个三元组，这样可以在有限的存储空间内最大化查询准确率。
核心问题
知识图谱通常非常大（包含上百万三元组），但用户的查询通常只涉及其中一小部分。系统需要从大图中挑选出最重要的K个三元组组成摘要，使得后续查询能在摘要上获得高准确率。

⚙️ 核心原理

1. APEX算法原理
APEX为每个三元组计算热度值H = e₁ × (r + smoothing) × e₂，其中e是实体热度，r是关系热度。
实体热度通过随机游走计算，关系热度通过查询频率统计。
然后选择top-K个三元组构成摘要。
2. APEX-N算法原理
APEX-N使用热扩散代替矩阵运算，更适合大规模图。
通过heat diffusion将查询的热量扩散到相邻实体，按热度排序选择top-K实体，再构建这些实体相关的三元组摘要。
3. 增量更新
系统使用增量排序算法，当新查询到来时，只需更新变化的三元组热度，而不是重新计算全部。大大提高了更新效率。

🛠️ 技术栈

Python 3

主要编程语言，简洁高效，适合科学计算和算法实现。

NumPy

数值计算库，提供高效的数组和矩阵操作。

SciPy

科学计算库，提供稀疏矩阵(CSR)实现，节省内存。

tqdm

进度条库，用于显示算法执行的进度。

CSR矩阵

Compressed Sparse Row，稀疏矩阵存储格式，高效存储图结构。

随机游走算法

带重启的随机游走，计算实体重要性。

🚀 入手点

快速运行命令

# 快速启动（使用MetaQA） cd code python main.py --kg MetaQA --method apex apex-n --percent-triples 0.0001 --n_users 1
# 生成查询并测试（DBpedia） python main.py --kg DBpedia --method apex apex-n --save-queries --percent-triples 0.000001 --n_users 1
# 查看日志结果 # 在 log.log 中搜索 "Ave Time" 和 "Ave Ave F1"

关键入口文件

# 入口：code/main.py
# 核心算法：code/src/apex.py (APEX, APEX_N函数)
# 知识图谱：code/src/base.py (KnowledgeGraph类)
# 查询处理：code/src/query.py (answer_query函数)

📁 支持的数据集

MetaQA

电影问答数据集，包含9万余条三元组。已包含在代码库中，可直接使用。适合快速测试。

DBpedia

百科全书知识图谱，需要从官网下载数据文件。数据较大（约600MB压缩）。

YAGO

马克斯·普朗克研究所的知识图谱，数据需要从官网下载。

Freebase

Google的知识图谱，已停止更新，代码支持但数据需要单独获取。

⚡ 关键参数

--percent-triples
摘要大小K = 知识图谱三元组数 × percent-triples。值越小摘要越紧凑。
--gamma
热衰减因子，控制历史热量的遗忘速度。值越小遗忘越快。
--gamma
知识图谱选择（MetaQA/DBpedia/YAGO/Freebase）。
--method
摘要算法选择（apex/apex-n）。
