📄 code/src/base.py

知识图谱基类和具体实现模块，包含YAGO、DBpedia、MetaQA、Freebase四种知识图谱的实现。该文件是整个系统的核心数据管理模块。

类和继承关系

KnowledgeGraph - 基础抽象类

所有知识图谱的基类，提供通用的三元组管理接口。

核心属性：

  • entities_ - 实体集合
  • relationships_ - 关系集合
  • triples_ - 三元组字典 {e1: {r: {e2}}}
  • entity_id_ / id_entity_ - 实体与ID的映射
  • csr_matrix_ - CSR稀疏矩阵（ adjacency matrix）
YAGO - YAGO知识图谱实现

继承自KnowledgeGraph，处理YAGO数据集格式。

DBpedia - DBpedia知识图谱实现

继承自KnowledgeGraph，处理DBpedia数据集格式。

MetaQA - MetaQA知识图谱实现

继承自KnowledgeGraph，处理MetaQA电影问答数据集。

特色：包含关系映射表，将原始关系转换为双向关系。

核心方法

  • add_triple(triple)

    添加三元组到知识图谱，自动维护实体ID和关系ID的映射。

  • csr_matrix()

    构建CSR稀疏矩阵，用于高效的图算法计算。

  • csr_matrix_indirect()

    构建无向图的CSR矩阵（邻接矩阵+转置）。

  • calc_csr_matrix_indirect_heat()

    构建用于热扩散的列随机矩阵。

  • load(head=None)

    加载知识图谱数据，从文件读取三元组。

  • model_user_pref(query_log)

    基于查询日志建模用户偏好，计算实体和三元组的热度值。

    • 调用链关系

    main.py └─> base.py (KG.load, KG.update_user) ├─> algorithms.py (query_vector, random_walk_with_restart) └─> path.py (数据路径配置)

    📚 术语解释（通俗版）

    三元组 (Triple) 表示知识图谱中的一条边，格式为(头实体, 关系, 尾实体)。���如��张三-朋友-李四。
    CSR矩阵 (Compressed Sparse Row) 一种稀疏矩阵存储格式，用三个数组存储非零元素，节省内存。像邻接表的升级版。
    热扩散 (Heat Diffusion) 一种图上的传播算法，类似热量在金属中传导，用于计算实体的重要性。
    随机游走 (Random Walk) 从一个节点随机跳到相邻节点的过程，用于分析图结构。
    大白话总结：base.py是整个系统的心脏，它管理所有的知识图谱数据。它把原始的三元组数据变成计算机可以高效处理的矩阵形式，并提供查询和更新的接口。不同的知识图谱（YAGO、DBpedia、MetaQA）有不同的数据格式，但都抽象成了统一的接口。
