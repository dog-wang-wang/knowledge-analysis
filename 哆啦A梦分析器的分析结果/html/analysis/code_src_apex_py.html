📄 code/src/apex.py

APEX和APEX-N摘要算法的核心实现文件。这是项目的核心算法模块，实现了自适应个性化知识图谱摘要。

核心算法

APEX - 基础版摘要算法

基于热衰减的增量式知识图谱摘要算法。

  • 输入：KG, K, query_log, gamma, diameter, alpha
  • 输出：F1分数列表、更新耗时列表
  • 原理：为每个三元组计算热度值H = e1 × (r + smoothing) × e2，然后选择top-K个三元组构成摘要
    • APEX-N - 改进版摘要算法

    基于热扩散的增量式摘要算法，更适合处理大规模图。

  • 输入：KG, K, query_log, gamma, diameter, alpha
  • 输出：F1分数列表、更新耗时列表
  • 原理：使用热扩散计算实体热度，按热度排序选择top-K实体，再构建摘要

    • 辅助函数

  • Heat_Diffuse(heat, KG, query, d, csr_indirect_matrices)

    热扩散函数，将查询的热量扩散到相邻实体。

  • heat_decay_array(h, gamma)

    热衰减函数，按gamma系数衰减热量。

  • binary_find(index_list, changed_index, H)

    二分查找，在排序列表中查找指定索引。

  • binary_insert(index_list, changed_index, H)

    二分插入，保持列表有序。

  • incremental_sort(index_list, changed_index_list, H_old, H_new)

    增量排序，高效更新有序列表。

  • construct_naive(KG, K, topic_eids)

    朴素方式构建摘要。

  • construct_complete(KG, K, index_list)

    完整方式构建摘要。

    • Summary类

    class Summary(KnowledgeGraph): - fill(triples, k): 填充三元组到摘要 - add_entity(topic_mid): 添加实体相关三元组 - add_random_triple(): 随机添加一个三元组

    调用链关系

    main.py └─> apex.py (APEX, APEX_N) ├─> base.py (KnowledgeGraph, Summary) └─> metrics.py (total_query_log_metrics, average_query_log_metrics)

    📚 术语解释（通俗版）

    摘要 (Summary) 知识图谱的一个子图，包含K个最重要的三元组，用来做快速查询。
    热度值 (Heat) 表示三元组重要性的数值，热度越高说明越可能是用户关心的。
    gamma 衰减因子 控制历史热量衰减的参数，值越小历史数据忘记得越快。
    增量更新 只更新变化的部分，而不是重新计算全部，提高效率。
    大白话总结：apex.py是APEX系统的核心发动机。它通过分析用户的查询历史，计算出哪些三元组最重要，然后挑选top-K个组成一个"小图谱"（摘要）。这样用户再查询时，可以先在小图谱里找答案，速度快很多。APEX-N是改进版，处理大图更高效。
