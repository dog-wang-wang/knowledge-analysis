📄 code/src/user.py

用户查询日志生成模块，根据主题实体和查询模式生成模拟的用户查询日志。

核心函数

query_log_by_mids(KG, topic_mids, n_queries_in_log, ...)

根据指定的主题实体MID生成查询日志。

query_log_by_topics(KG, topics, n_mids_per_topic, n_queries_in_log, ...)

根据高层主题生成查询日志。

generate_queries_by_mid(KG, topic_mid, n_mid_queries, start_qid)

为指定的主题实体生成多个查询。

generate_queries_by_topic(KG, topic, n_topic_queries, n_topic_mids)

为指定主题生成查询。

randomize_log(KG, query_log, random_query_prob, shuffle)

随机化查询日志，可以添加随机查询或打乱顺序。

辅助统计函数

  • reuse(query_log) - 计算查询日志的重用率
  • entity_counts(query_log) - 统计每个主题实体的查询次数
  • predicate_counts(query_log) - 统计每个关系的查询次数

    • 调用链

    user.py (query_log_by_mids)
    └─> query.py (generate_query)
    └─> base.py (KG查询)

    📚 术语解释

    查询日志 (Query Log)用户一系列查询的记录，包含主题实体和查询关系。
    主题MID (Topic MID)查询的主题实体标识符。
    重用率 (Reuse)重复查询占总查询的比例。
    大白话总结：user.py负责生成"假"的用户查询。它根据知识图谱中的实体，随机生成一些问题，作为测试摘要算法的输入。生成的查询会模拟真实用户的行为模式。
