📄 code/src/query.py

查询处理模块，负责查询的加载、保存、生成和答案计算。查询格式遵循WebQSP数据集标准。

WebQSP查询格式

{
    "QuestionId": "q1",
    "Parse": {
        "TopicEntityMid": "m.0xxx",
        "TopicEntityName": "实体名",
        "InferentialChain": ["关系1", "关系2"],
        "Constraints": [...],
        "Answers": [{"AnswerType": "Entity", "AnswerArgument": "..."}]
    }
}

核心函数

answer_query(KG, query)

在知识图谱中查询答案，返回符合条件的实体集合。

generate_query(KG, topic_mid, chain_len, qid, ...)

生成合成查询，随机选择路径构建查询。

load_question(query_fname)

从JSON文件加载查询。

save_question(question, query_fname)

保存查询到JSON文件。

load_questions_from_dir(query_dir)

从目录加载所有查询。

load_questions_from_file(query_dir, fname)

从文件加载指定查询ID列表的查询。

辅助函数

  • check_question(question) - 检查查询格式是否正确
  • get_name(mid, entity_names) - 获取实体的可读名称
  • is_synthetic_query(query) - 判断是否为合成查询
  • load_qids(fname) - 从文件加载查询ID列表

    • 📚 术语解释

    WebQSP一个知识库问答数据集标准格式。
    InferentialChain查询的推理链，即经过的关系序列。
    TopicEntity查询的主题实体。
    大白话总结：query.py负责"问问题"和"找答案"。它定义了查询的格式（JSON），并且提供在知识图谱中查找答案的功能。
