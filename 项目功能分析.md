# APEX 项目功能分析报告

## 1. 项目概述
APEX 是一个用于知识图谱（Knowledge Graph）摘要生成的实验框架，核心目标是根据用户的实时查询行为，动态生成并维护一个个性化的、小规模的知识子图，以确保该子图能够覆盖用户最关心的信息。

## 2. 核心功能模块

### 2.1 数据集管理 (Data Management)
- **多源支持**：内置了对 YAGO, DBpedia, Freebase 和 MetaQA 的适配器。
- **高效存储**：通过 `KnowledgeGraph` 类管理数百万级的三元组，并构建 CSR 稀疏矩阵索引。

### 2.2 用户与查询模拟 (User & Query Simulation)
- **兴趣建模**：基于主题（Topic）或特定实体（MID）模拟用户的关注点。
- **流式查询**：模拟用户在不同时间戳下提出的查询请求，形成 Query Log。

### 2.3 摘要生成算法 (Summarization Algorithms)
- **APEX**：计算三元组（Subject, Relation, Object）的综合热度值，通过增量排序维护 Top-K 摘要。
- **APEX-N**：专注于实体的热度扩散，适用于对图结构要求不同的场景。
- **热扩散机制**：利用 Laplacian 矩阵及其变体实现热量在图上的衰减与传播。

### 2.4 自动化评估 (Evaluation)
- **指标计算**：自动计算每一轮更新后的 F1-score、Precision 和 Recall。
- **性能监控**：记录算法的更新耗时（Update Time），评估系统实时性。

## 3. 业务流程
1. 加载全量知识图谱并初始化稀疏矩阵。
2. 为指定用户生成或加载查询日志。
3. 针对每一条查询：
   - 更新系统内的热度分布（Heat Diffusion）。
   - 增量更新摘要内容（Incremental Update）。
   - 计算当前摘要在下一条查询上的表现。
4. 输出平均性能统计结果。
