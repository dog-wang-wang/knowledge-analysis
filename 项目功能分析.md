# APEX² 项目功能分析报告

## 1. 核心目标
APEX²（Adaptive and Extreme Summarization for Personalized Knowledge Graphs）项目的核心目标是为超大规模知识图谱（KG）提供一种**自适应**且**极端**的摘要算法。它能在保留用户感兴趣信息的同时，将图谱规模缩减到极小比例（如 0.0001%）。

## 2. 功能模块划分

### 2.1 数据处理与加载 (Data Management)
- **多数据源支持**: 支持 MetaQA, DBpedia, YAGO 三种主流知识图谱。
- **解析器 (Parser)**: 针对不同格式（如 `.nt`, `.tsv`, `.txt`）提供统一的三元组提取逻辑。
- **查询模拟**: 支持 WebQSP 格式的查询解析，模拟真实用户与图谱的交互。

### 2.2 个性化摘要算法 (Core Algorithms)
- **APEX / APEX-N**: 基于热扩散（Heat Diffusion）和随机游走（RWR）的自适应摘要逻辑。
- **热扩散模拟**: 利用 `scipy.sparse.linalg.expm_multiply` 高效计算大图上的热传导，模拟实体相关性。
- **懒惰贪婪优化 (Lazy Greedy)**: 在 `heap.py` 中实现，通过堆结构优化子模函数选择，提升摘要效率。

### 2.3 用户行为模拟 (User Profiling)
- **兴趣漂移模拟**: `user.py` 模拟用户兴趣随时间变化的过程，验证算法的自适应性。
- **反馈闭环**: 根据用户查询结果调整实体权重，实现动态个性化。

### 2.4 性能评估 (Evaluation Metrics)
- **F1 分数**: 衡量摘要图谱对原始查询的覆盖程度和准确性。
- **时间效率**: 记录大规模矩阵运算和摘要生成的耗时。

## 3. 关键业务流程
1. **初始化**: 加载原始 KG，构建稀疏邻接矩阵。
2. **种子生成**: 根据初始用户查询（或随机生成）确定初始“种子实体”。
3. **扩散计算**: 使用热扩散算法计算全图实体与种子的关联分值。
4. **摘要提取**: 利用贪婪策略选择 Top-K 重要的三元组构成摘要图。
5. **迭代更新**: 用户发起新查询 -> 算法根据反馈更新权重 -> 重新生成摘要。

## 4. 总结
该项目是一个高度工程化的科研项目，将前沿的图论算法（热扩散、RWR）与工业级稀疏矩阵计算相结合，解决了大规模个性化知识服务中的存储与检索效率问题。
