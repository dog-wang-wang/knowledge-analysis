# APEX 项目功能分析报告

## 1. 项目概述
**APEX** (Adaptive Personalized Knowledge Graph Summarization) 是一个专注于知识图谱个性化摘要的研究性项目。其核心目标是根据用户的历史查询行为，从庞大的知识图谱中自动提取出一个较小的、最能代表用户兴趣的子图（摘要）。

## 2. 核心功能模块

### 2.1 知识图谱表示与加载 (`code/src/base.py`)
- **统一接口**：定义了 `KnowledgeGraph` 基类，为不同的知识图谱（YAGO, Freebase, DBPedia, MetaQA）提供统一的操作接口。
- **高效存储**：利用 `scipy.sparse.csr_matrix` 将图结构转换为稀疏矩阵，支持高效的图计算（如邻接矩阵、转置矩阵、转移矩阵计算）。
- **数据适配**：实现了针对不同数据集格式（如 `.gz` 压缩的 RDF 格式、文本格式）的加载器。

### 2.2 个性化摘要算法 (`code/src/apex.py`)
- **APEX 算法**：项目的主算法，通过分析用户查询日志，计算实体和三元组的权重。
- **APEX-N 算法**：一种改进或变体算法，用于对比实验。
- **动态更新机制**：支持根据新的查询实时更新摘要，反映用户兴趣的漂移。

### 2.3 用户偏好建模 (`code/src/algorithms.py`, `code/src/user.py`)
- **随机游走 (RWR)**：使用带重启的随机游走算法在知识图谱上扩散用户查询的信号，从而发现潜在感兴趣的关联实体。
- **查询模拟**：在缺乏真实用户数据时，通过 `user.py` 模拟生成具有主题聚集性的用户查询日志。

### 2.4 实验评估框架 (`code/main.py`, `code/src/metrics.py`)
- **自动化测试**：`main.py` 作为入口，支持配置实验参数（如摘要大小 K、模拟用户数、衰减因子等）。
- **多维度评价**：支持计算 F1 分数、准确率以及算法运行的响应时间。

## 3. 技术栈
- **编程语言**：Python 3
- **数学与图计算**：NumPy, SciPy (Sparse Matrix)
- **数据处理**：Standard Libraries (json, gzip, re, argparse)
- **实验工具**：Logging, pdb (调试)
