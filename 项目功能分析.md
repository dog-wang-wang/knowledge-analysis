# 项目功能分析 - APEX (Adaptive Personalized Knowledge Graph Summarization)

本项目名为 **APEX**，全称为 **Adaptive Personalized Knowledge Graph Summarization**（自适应个性化知识图谱摘要）。其核心功能是根据用户的查询历史，动态、实时地为大型知识图谱（KG）生成一个精简的摘要（子图），以在资源受限的情况下最大化问答任务的准确性。

## 核心功能模块

### 1. 知识图谱管理 (src/base.py)
*   **统一接口**: 提供 `KnowledgeGraph` 基类，定义了实体、关系、三元组的基本操作。
*   **多数据集支持**: 实现了对 **Freebase**, **YAGO**, **DBPedia**, 和 **MetaQA** 四种主流知识图谱数据集的加载和解析。
*   **高效表示**: 使用 `scipy.sparse.csr_matrix` 存储图结构，支持快速的图计算（如邻接矩阵、转置矩阵、热度扩散矩阵等）。

### 2. 用户查询模拟与管理 (src/query.py, src/user.py)
*   **查询解析**: 遵循 WebQSP 格式解析 JSON 查询，处理查询路径（Inferential Chain）和约束。
*   **模拟日志生成**: 可以根据指定的 Topic MIDs 或主题为特定用户生成查询序列，模拟用户的兴趣偏好。
*   **自动答题**: 在给定的知识图谱（完整图或摘要图）上执行查询路径，获取答案。

### 3. APEX 摘要算法实现 (src/apex.py)
*   **APEX (Complete)**: 
    *   **热度扩散 (Heat Diffusion)**: 基于当前查询动态更新实体和关系的热度。
    *   **增量排序 (Incremental Sort)**: 使用高效的二分查找和插入算法，仅更新受影响的三元组权重，避免全量重排。
    *   **摘要构建**: 根据权重排名，在满足三元组数量限制 $K$ 的前提下，提取最优子图。
*   **APEX-N (Naive/Entity-based)**:
    *   简化版算法，主要基于实体的热度来选择三元组。

### 4. 优化与辅助模块 (src/heap.py, src/algorithms.py)
*   **Lazy Greedy 堆优化**: `Heap` 类实现了惰性贪婪搜索（Lazy Greedy），用于在静态或半静态场景下高效筛选高价值三元组。
*   **图算法**: 实现了带重启的随机游走（Random Walk with Restart）和查询向量计算。

### 5. 评估指标 (src/metrics.py)
*   **准确率 (Precision)**, **召回率 (Recall)**, **F1 分数**: 针对单个查询或整个查询日志进行评估。
*   **性能对比**: 统计摘要图在问答任务上与原始图的一致性。

### 6. 主程序入口 (main.py)
*   提供命令行工具，支持配置数据集（KG）、用户数、查询数、摘要大小约束 $K$、衰减因子 $\gamma$ 等参数。
*   执行模拟实验，并输出各算法的平均 F1 分数和更新耗时。
