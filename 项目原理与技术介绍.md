# APEX 项目原理与技术介绍

## 1. 核心原理：基于热扩散的自适应摘要
APEX (Adaptive Personalized Knowledge Graph Summarization) 的核心思想是**将用户的查询兴趣建模为知识图谱上的能量传播**。

### 1.1 热扩散模型 (Heat Diffusion Model)
- **初始能量**：当用户查询某个实体（Topic Entity）或关系时，该实体被赋予初始“热度”。
- **传播机制**：热度沿着图的边向邻居节点扩散。项目利用 Laplacian 矩阵的变体来模拟这一物理过程。
- **时间衰减**：引入衰减因子 $\gamma$（通常为 0.5）。随着时间推移，旧查询带来的热度会指数级下降，从而保证摘要的**时效性**。

### 1.2 个性化摘要维护
- **三元组评分**：一个三元组 $(s, r, o)$ 的得分由头实体 $s$、关系 $r$ 和尾实体 $o$ 的当前热度共同决定：
  $$Score(s, r, o) = Heat(s) \times (Heat(r) + \text{smoothing}) \times Heat(o)$$
- **Top-K 提取**：系统实时维护得分最高的 $K$ 个三元组，构成当前用户的个性化知识摘要。

## 2. 关键技术栈
- **数值计算**：
    - **NumPy**: 用于处理高维向量运算。
    - **SciPy (csr_matrix)**：由于知识图谱极其稀疏，项目大量使用 CSR（Compressed Sparse Row）格式进行高效的矩阵乘法，加速热扩散计算。
- **算法优化**：
    - **增量排序 (Incremental Sort)**：在每轮查询后，仅对受影响的节点进行局部重新排序，而非全量重排，显著降低了计算复杂度。
- **仿真系统**：
    - **随机游走 (Random Walk)**：用于初始化用户兴趣或补充摘要内容。
    - **用户日志模拟**：模拟真实场景下用户对特定领域（如电影、历史）的持续关注。

## 3. 入手指南
如果您想深入研究或修改本项目，建议按以下顺序阅读代码：

1.  **`code/main.py`**：程序的“指挥部”。在这里你可以看到如何配置参数（如 `--kg`, `--gamma`）以及完整的实验循环。
2.  **`code/src/apex.py`**：算法的核心。重点关注 `Heat_Diffuse` 函数（热量如何传播）和 `APEX` 类（摘要如何更新）。
3.  **`code/src/base.py`**：数据结构定义。了解 `KnowledgeGraph` 是如何通过字典和稀疏矩阵双重索引数据的。
4.  **`PKG_EXP/`**：数据目录。查看 `MetaQA` 的 `kb.txt` 了解原始知识图谱的存储格式。

## 4. 总结
APEX 不仅仅是一个静态的子图提取工具，它更像是一个**具有记忆的动态过滤器**，能够随着用户兴趣的迁移而自动演化其维护的知识库。
