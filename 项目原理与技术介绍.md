# APEX 项目原理与技术介绍：把大图谱变“小”的艺术

## 1. 这项目到底是干啥的？（通俗版）
**APEX** 的核心任务就是：**从海量数据中，捞出你最关心的那一小块知识。**

想象你面前有一张覆盖全球的巨大社交网络图，你只想看和你、你的朋友、以及你们喜欢的明星相关的部分。APEX 就像一个智能的“聚光灯”，它会根据你查过什么，自动把聚光灯打在最相关的节点上，并把没用的背景调暗。

## 2. 核心原理：像“热量传导”一样的兴趣追踪

### 2.1 兴趣激发与扩散（注入热量）
- **激发 (Excitation)**：当你搜一个词，这个节点就“烧”起来了（获得热量）。
- **扩散 (Propagation)**：热量会顺着关系线传给邻居。比如“周杰伦”很烫，那“昆凌”和“范特西”也会跟着变暖。
- **衰减 (Decay)**：人的兴趣会变。时间久了，旧的兴趣节点就会慢慢“冷却”，确保系统只关注你现在的热点。

### 2.2 怎么决定谁进“精选集”？
系统给每个三元组（实体-关系-实体）算分：
$$总分 = 头实体的热度 \times 关系的权重 \times 尾实体的热度$$
分越高，说明这个知识点对你越重要，就越应该被放进摘要里。

## 3. 它是怎么跑得飞快的？（技术狠活）

### 3.1 稀疏矩阵加速
项目把几千万个关系变成了数学里的“稀疏矩阵”。扩散热量的时候，不再是一个一个点去爬，而是直接做高效的矩阵乘法，把速度提升了成百上千倍。

### 3.2 增量排序（只算变动的那一丁点）
这是 APEX 的杀手锏。当你有新查询时，它**不会**重算整张图，而是像记账一样，只更新受影响的那些点的排名。
- **普通做法**：每次都给全校学生重新排名（慢）。
- **APEX 做法**：只看刚考完试的那几个学生，把他们插入到正确的位置（极快）。

## 4. 快速上手指南

### 4.1 准备环境
```bash
# 安装最核心的三个工具：
# numpy (算数学), scipy (算大矩阵), tqdm (看进度条)
pip install numpy scipy tqdm
```

### 4.2 执行 MetaQA 实验（最推荐的快速入门）
```bash
# 先进入代码目录
cd code

# 跑一个 MetaQA 的快速示例
# --kg: 指定数据集
# --method: 跑哪种算法 (apex 是我们的主打)
# --percent-triples: 摘要的规模，0.0001 表示只提取万分之一的精华
python main.py --kg MetaQA --method apex apex-n --percent-triples 0.0001 --n_users 1
```

## 5. 关键文件地图
- **大脑 (算法实现)**：`code/src/apex.py` —— 所有的热量扩散、增量排序逻辑都在这。
- **骨架 (数据结构)**：`code/src/base.py` —— 定义了怎么加载 YAGO、DBpedia 这种庞然大物。
- **指挥部 (运行入口)**：`code/main.py` —— 负责模拟用户行为、统计准确率（F1 分数）和运行时间。
