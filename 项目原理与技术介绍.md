# 项目原理与技术介绍 - APEX

## 1. 项目背景
在知识图谱（KG）问答系统中，完整的知识图谱往往包含数亿个三元组，这对存储和计算资源提出了巨大挑战。**APEX** 旨在通过“摘要（Summarization）”技术，提取出一个与用户兴趣高度相关的微型子图，在保证问答准确率的同时，极大地降低资源消耗。

## 2. 核心原理

### 2.1 个性化建模 (Personalization)
项目认为用户的兴趣可以通过其历史查询日志（Query Log）来推断。每个查询涉及的实体（Topic Entity）和关系（Relation）是用户关注的中心。

### 2.2 自适应更新 (Adaptive Update)
用户兴趣随时间变化。APEX 采用了一种“热度衰减”模型：
- **热度增加**: 当用户发起新查询时，相关实体和三元组的热度值增加。
- **时间衰减**: 随着时间推移，旧的查询对当前兴趣的影响按因子 $\gamma$ 指数级衰减。
- **公式**: $H_{new} = H_{old} \times \gamma + \Delta H_{current}$

### 2.3 热度扩散算法 (Heat Diffusion)
热度不仅停留在查询直接提到的实体上，还会通过图的结构向邻居扩散。项目使用稀疏矩阵运算实现了一阶或多阶的热度扩散，从而挖掘出潜在相关的上下文信息。

### 2.4 增量排序优化 (Incremental Sorting)
这是 APEX 能够实现实时更新的关键。
- 传统的做法是每次更新后对所有三元组重新排序，复杂度极高。
- APEX 只针对受当前查询影响的实体和三元组进行局部更新，并利用增量排序算法（基于二分查找和插入）维护有序列表，显著降低了计算开销。

## 3. 技术栈
- **Python 3.x**: 核心开发语言。
- **SciPy (Sparse Matrix)**: 用于处理大规模稀疏图结构，实现高效的矩阵乘法和转置。
- **NumPy**: 用于向量化计算和数值处理。
- **Tqdm**: 提供进度条展示，优化实验体验。
- **Argparse & Logging**: 提供规范的命令行交互和运行日志记录。

## 4. 入手点建议

如果您想深入了解本项目，可以按以下顺序阅读代码：

1.  **`code/main.py`**: 程序总入口。了解如何加载数据集、模拟用户行为以及调用实验流程。
2.  **`code/src/base.py`**: 了解底层知识图谱的存储结构（`KnowledgeGraph` 类）以及如何支持不同的数据集（YAGO, Freebase 等）。
3.  **`code/src/apex.py`**: **最核心的算法实现**。重点查看 `APEX` 函数，理解它是如何执行热度扩散和增量更新的。
4.  **`code/src/metrics.py`**: 了解评估摘要质量的标准（F1, Precision, Recall）。

## 5. 快速开始
参考 `main.py` 的参数说明，您可以尝试运行以下命令进行简单的实验：
```bash
python code/main.py --kg MetaQA --method apex --n-queries 50 --percent-triples 0.001
```
这将使用 MetaQA 数据集，为每个用户模拟 50 条查询，并提取 0.1% 的三元组作为摘要。
