# APEX 项目原理与技术介绍

## 1. 核心原理：个性化热量扩散机制
APEX 的基本思想是将用户的查询行为视为对知识图谱（KG）的“能量注入”。

### 1.1 热量注入与扩散
- **激发 (Excitation)**：当用户查询某个实体 $E$ 时，该节点获得初始热量。
- **传播 (Propagation)**：热量沿着图中的边向外扩散。APEX 使用一种改进的随机游走算法，考虑了节点间的连接强度。
- **衰减 (Decay)**：为了捕捉用户兴趣的漂移，系统会对所有节点的热量进行周期性衰减。

### 1.2 三元组评分
一个三元组是否应该进入摘要，取决于它的两个端点实体是否“热”以及关系本身的重要性：
$$Score = Heat(head) \times RelationshipWeight \times Heat(tail)$$

## 2. 关键技术实现

### 2.1 高性能图运算
- **稀疏矩阵**：项目将整个知识图谱表示为 CSR (Compressed Sparse Row) 格式的稀疏矩阵。扩散过程被简化为矩阵与向量的乘法运算，极大地提升了处理千万级三元组的速度。
- **线性代数加速**：利用 `scipy.sparse` 进行矩阵转置、归一化及迭代运算。

### 2.2 增量更新算法
- **避免全量计算**：APEX 不需要每次都重新扫描整个图。它记录了受当前查询影响的“活跃子图”范围。
- **增量排序**：维护一个有序的候选池，仅更新受影响项的排名，使得摘要更新达到了毫秒级的响应速度。

### 2.3 实验仿真环境
- **多图谱支持**：内置了 YAGO, Freebase, DBPedia 等大型通用知识图谱的解析器。
- **用户行为模拟**：实现了基于 LDA 主题模型的查询生成器，能够模拟具有特定兴趣偏好的用户。

## 3. 快速入手指南

### 3.1 环境准备
```bash
pip install numpy scipy tqdm
```

### 3.2 运行核心实验
```bash
# 使用 MetaQA 数据集运行 APEX 算法
python code/main.py --kg MetaQA --method apex --n-queries 100
```

### 3.3 关键代码位置
- **算法核心**：`code/src/apex.py` (包含 `APEX` 和 `APEX_N` 函数)
- **图数据结构**：`code/src/base.py` (包含 `KnowledgeGraph` 类)
- **数学运算**：`code/src/algorithms.py` (包含 RWR 算法实现)
